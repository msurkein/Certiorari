Label mappings

Templates decide the certificate label, matched by issuer. First matching rule wins; otherwise the built-in CN - OU1 - OU2 … format is used.

Template reference (Liquid)

Variables (from the Subject):

  • cn — first CN · ou — array, 0-indexed: ou[0], ou[1], …
  • o, c, l, st, email — first value of each
  • issuer.cn, issuer.o, dn (full subject string)

Filters:

  • upcase, downcase, capitalize, strip
  • slice: a, b (substring) · append: 'x' · prepend: 'x' · default: 'x'
  • skip: n (drop first n chars) · take: n (keep first n chars)

Examples:

  • {{ ou[1] | upcase }} - {{ ou[3] | upcase }} - {{ ou[4] | upcase }}
  • {{ ou[0] }} - {{ ou[4] | skip: 5 }} - {{ ou[1] }}
